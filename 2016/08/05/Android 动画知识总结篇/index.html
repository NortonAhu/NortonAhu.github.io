动画分类

在 Android 体系中提供给我们提供了两种动画框架：View Animation 和 Property Animation 视图动画和属性动画。其中属性动画是在 Android 3.0 之后才引入到 Android 框架中的，所以如果想兼容 3.0 之前的Android版本需要添加第三方lib库。除了这两种动画外还有一种 Drawable 动画或者叫Frame Aniamtion，因为是对图片的切换造成动画的效果所以动画分类一般也可以不说它。

Drawable 动画

Drawable Animation 实现的原理是顺序播放事先做好的图像，这种动画一种渐变动画。类似于老式的动画制作都是一张张画出来然后快速翻页就动起来了。要注意的是这种类型的动画认为是drawable，所以该动画的XML文件既不能放在anim也不能放在animator文件夹中，这两个文件夹的区别后面会提到的。

Tween 动画

通过对场景里的对象不断做图像变换(平移、缩放、旋转、透明度)产生动画效果，即是一种渐变动画。其中原理主要是对 view 做矩阵转换操作。补间动画是可以添加动画的侦听事件这样就比较好控制动画操作。这里如果使用XML方式来创建动画需要在res目录下建立一个anim文件夹放xml文件。下面介绍的是渐变动画的几个主要类型。

Scale Animation

表示简单的视图的缩放功能，参数有

  • duration 动画从开始到结束持续的时长，单位为毫秒；
  • fromXScale 动画开始时X坐标上的缩放尺寸；
  • toXScale 动画结束时X坐标上的缩放尺寸；
  • fromYScale 动画开始时Y坐标上的缩放尺寸；
  • pivotX 缩放时的固定不变的X坐标，一般用百分比表示，0%表示左边缘，100%表示右边缘；
  • pivotY 缩放时的固定不变的Y坐标，一般用百分比表示，0%表示顶部边缘，100%表示底部边缘。

前四个属性，0.0表示缩放到没有，1.0表示正常无缩放，小于1.0表示收缩，大于1.0表示放大，后面的两个其实也可以课 translate 一样加上 ％p 的参数也是表示父类控件的比例。

set

我们在anim文件夹中创建动画时默认是创建 set ，我们 duration 和 repeatCount 等属性也是属于 set 里面的属性所以我们可以认为 set 是动画外面嵌套的层，通过 set 我们可以对动画进行组合。

水平移动缩小的动画：

set 属性除了设置 duration 剩下的参数是：

  • android:detachWallpaper 设置是否在壁纸上运行，只对设置了壁纸背景的窗口动画(window animation)有效。设为true，则动画只在窗口运行，壁纸背景保持不变;
  • android:fillAfter 设置为true时，动画执行完后，View会停留在动画的最后一帧；默认为false；如果是动画集，需在标签中设置该属性才有效;
  • android:fillBefore 设置为true时，动画执行完后，View回到动画执行前的状态，默认即为true;
  • android:fillEnabled 设置为true时，android:fillBefore的值才有效，否则android:fillBefore会被忽略;
  • android:repeatCount 设置动画重复执行的次数，默认为0，即不重复；可设为-1或infinite，表示无限重复;
  • android:repeatMode 设置动画重复执行的模式，可设为以下两个值其中之一：
  • restart 动画重复执行时从起点开始，默认为该值;
  • reverse 动画会反方向执行;
  • android:startOffset 设置动画执行之前的等待时长，毫秒为单位；重复执行时，每次执行前同样也会等待一段时间;android:zAdjustment 表示被设置动画的内容在动画运行时在Z轴上的位置，取值为以下三个值之一：
  • normal 默认值，保持内容在Z轴上的位置不变
  • top 保持在Z周最上层;
  • bottom 保持在Z轴最下层;
  • android:interpolator 设置动画速率的变化，比如加速、减速、匀速等，需要指定Interpolator资源，后面再详细讲解。
  • 标签还有个android:shareInterpolator属性，设置为true时则可将interpolator应用到所有子元素中
    以上基本就将渐变动画主要内容介绍，关于插值器属性动画也有后面会单独介绍的。

小结：set 表示对 Tween Animation 做动画集操作。

Property 动画

属性动画是真正意义上对对象的属性进行操作。因此它和视图动画的区别有以下三个方面：

  • API 不同属性动画是在 Android 3.0 后引入到 Android 框架中，所以在 3.0 之前是不可以使用的。
  • 属性动画是真正意义上的对对象的属性进行修改达到动画的效果的，而视图动画确实修改了 View 的绘制方法，最明显的不回改变 View 的点击区域；
  • 属性动画是对任何对象都可以操作的，但是视图动画操作的对象只能是 View。

因此属性动画可以定义为在一定的时间内按照一定的规律来改变对象的属性的动画。

从上面的属性动画的定义我们可以看出我们对对象的属性进行操作这就是说我们要对象提供对象的 get/set 方法，这也是对象使用属性动画的必要条件。

  • object必须要提供setXxx方法，如果动画的时候没有传递初始值，那么还要提供getXxx方法，因为系统要去拿xxx属性的初始值（如果这条不满足，程序直接Crash）；
  • sobject的setXxx对属性xxx所做的改变必须能够通过某种方法反映出来，比如会带来ui的改变啥的（如果这条不满足，动画无效果但不会Crash）

插值器和估值器原理

在介绍插值器和估值器的时候需要介绍下 google 官方文档中关于 Property Animation 的知识。
属性动画六个要素：

  1. duration：动画的持续时间，默认设置是300ms；
  2. TimeInterPloation:用于定义动画变化率的接口；
  3. TypeEvaluator:用于定义属性值计算的接口，系统自带的 int／float／color等类型
  4. AnimationSets:动画集合，可以设置顺序；
  5. Frame refrsh delay：用于定义动画的刷新频率，默认的刷新频率是10ms每帧，但是这个时间最终是由手机当时的CPU繁忙程度和系统性能决定的；
  6. repeat count and behavior：定义动画的重复次数和行为的（类似于视图动画中的model）。

Step 1： 计算流失时间 fraction

public Integer evaluate(float fraction, Integer startValue, Integer endValue) {
int startInt = startValue;
return (int)(startInt + fraction * (endValue - startInt));
}

通过计算后得到 0 ＋ 40*0.15＝6
最后得到6就是这一帧该移动的X方向的距离。

上面介绍的是主要是的实现过程，我们现在来分析几个重要步骤
插值器
本身是时间函数，定义了动画的变化规律（提前或者延迟默认执行的时间来达到相应的效果正如前面的介绍的那样加速减速效果）
使用方法就是实现getInterpolation(float input)方法用来计算流失时间的插值器
流失时间是［0，1］计算得到的插值时间<0反方向直行，［0，1］预计方向直行，>1 超出正方向执行。

可以通过自定义不同的插值器实现出各种非常奇特的插值器出来。
小结：
a. 插值器是通过改变动画执行的时间点效果，提前或者延迟动画的执行；
b. 插值器只是针对当前动画的时间因子操作，并不对动画的轨迹进行操作。

这些就是Step1 到Step2的过程结束。
估值器
估值器是用来决定属性的计算方式，最终是用反射机制来实现对属性的变化设置
public interface TypeEvaluator {
public T evaluate(float fraction, T startValue, T endValue);
}
这个类是泛型的所以可以对属性的（get/set）值进行操作。
系统已经提供了 int、float、color的几种属性设置。

基本模式都是Result ＝ x0+(x1-x-)*fraction;
