Styles And Themes 详解

写在前面的话

 其实我们在写 Andorid 应用中打交道最多就是布局文件，我们有时为了某个布局文件调试的满头大汗。巧妙的利用 Android Style 可以帮助我们快速完美的解决一些布局上的风格问题，说这些感觉我很懂似得，其实我也不懂啦，只是想学习记录下。

读懂 Style API

 API 文档是我们了解一个东西的入口，所以我们刚开始就开始研究下 Style API 的风格如何。

风格和主题 (Styles and Themes)

 什么是 Style ? 其实它是指我们 View 和 Window 外观和风格的一组属性值的集合。
他被定义在 XML 文件中这样它就独立于布局文件存在。可以被多个布局文件复用了。一句话总结来说就是它可以让你的设计从内容中分离出来，类似于网页设计的级联表使用。(所有例子都是从 Andorid 官方获得)

—->

与 Style 相关联的是 Theme 这个属性主要用于 Activity 和 Application。一般来说在 AndroidMainfest Application 节点定义了 style 会被应用到整个主题上去。同理当一个 Activity 中使用了一个 Theme 的话这个布局文件中所有的 View 都会应用这个 Theme 中的属性值当然前提是 View 是支持这个属性的(具体该怎么查后面会讲到的)。

Style 值

 Style 主要是就是各种值的设置，有些值我们很了解 比如说 layout_width,padding 什么的，但是也有很多属性我们不知道，这就需要我们去查，怎么查看文档,比如说 TextView XML，如果想看完整的属性值的文档查R.attr 当然不是所有组件都是支持里面的属性的。

将 Style 应用到控件和主题上

有两种方式可以应用：

  • 直接将 Style 应用到控件上面去；
  • 使用 Application 或者 Activity 中的总体属性设置。

根据系统的版本选择T heme

 新版本都会有一些新的属性风格啥的，这里肯定要根据版本进行风格转换了。其实就是风格适配。平时看到的 res/values-v11 这些都是兼容性的方案具体的代码例子就不列举了。具体实例可以参考 Android 提供的资源文件 R.styleable.theme 和 Providing Resources 这两块内容能够提供主题的规范属性和能共提供的资源逻辑

使用系统自带的风格和主题资源(Using Platform Styles and Themes)

 Android 平台提供了大量的风格和主题资源文件，你可以查阅 R.style 其实谷歌官方体检一种更好的方式来查看 Android 平台提供的风格文件就是查看源文件，源文件主要有以下两个：

  • Android Styles (style.xml)
  • Android Themes (themes.xml)

可以根据这些定义的原始属性值来实现你自己的主题，比如 Dialog 的主题 就可以查看 Theme.dialog 属性设置，更多详情请查看 Style Resource

更多关于 View 的风格属性可以查看 R.attr
